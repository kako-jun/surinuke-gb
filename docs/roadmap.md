# ロードマップ

## 近い将来に届けたいこと

### 難易度の上昇

現在は一定のペースで障害物が降ってきますが、スコアが上がるにつれて落下速度や出現頻度が増していく仕組みを入れたいと考えています。序盤は余裕をもって遊べて、後半になるほど緊張感が増していく。「あとちょっとで新記録だったのに」という体験が、リプレイ意欲を生みます。

### 複数の障害物が同時に降ってくる

現在は画面上に障害物が1つだけですが、これを複数同時に降らせることで、回避の判断がより複雑になります。「左を避けたら右にもう1つ来ていた」という瞬間の緊張感を作りたい。ただし、同時に降らせすぎると理不尽になるので、回避可能な配置を保証する仕組みが必要です。

### タイトル画面

起動直後にゲームが始まる現在の仕様は手軽ですが、「すり抜けGB」のタイトルロゴとハイスコア表示があると、ゲームとしての完成度が一段上がります。ボタンを押すとゲーム開始、という一般的なフローを入れることで、最初に遊ぶ人にも「これはゲームだ」と伝わります。

### ハイスコアの保存

電源を切ってもハイスコアが残るようにしたいと考えています。「前回の自分を超える」という目標が生まれ、繰り返し遊ぶ理由になります。ゲームボーイのバッテリーバックアップ（SRAM）を使った保存を想定しています。

## もう少し先の展開

### 障害物の種類を増やす

直線的に落ちてくるだけでなく、斜めに動く障害物や、途中で停止してから再び落ちてくる障害物など、バリエーションを持たせることで飽きにくくなります。見た目も変えることで「次は何が来るか」という期待感が生まれます。

### ステージ制

一定スコアごとに壁の幅が変わったり、背景が変化したりするステージ制を導入できると、ゲームに展開が生まれます。「ステージ3まで行けた」という達成感が、スコアとは別の目標を与えます。

### 音楽

現在は効果音のみですが、ゲームボーイの音源チャンネルを使ったBGMがあると、ゲームの雰囲気が格段に良くなります。緊張感のあるループBGMが、回避アクションのテンポを引き立てます。

### 2人対戦

通信ケーブルを使った2人対戦は、ゲームボーイならではの体験です。同じ画面で同時にプレイし、先にゲームオーバーになった方が負け。実現の難易度は高いですが、ゲームボーイ開発の面白さを深堀りするテーマとして魅力があります。
